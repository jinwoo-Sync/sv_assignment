# PERF — 성능 측정 및 요구사항 검증 보고서

본 문서는 `요구사항.pdf`에 명시된 모든 시스템 요구사항(성능, 안정성, 확장성)의 충족 여부를 검증하기 위한 측정 결과 및 병목 분석 내용을 기록한다. 현재 버전에서는 `epoll` 기반 비동기 I/O가 대규모 연결(N=50) 환경에서 요구 스펙을 만족하는지 확인한다.

## 2. 측정 환경 및 시나리오

### 2.1 테스트 환경
- **OS**: Linux (Docker Desktop / Docker Engine)
- **Controller**: 1 인스턴스
- **Agent**: 1 ~ 50 인스턴스
- **전송 주기**: 각 Agent당 2Hz (500ms 고정)

### 2.2 측정 시나리오
- **시나리오 A (1v3)**: 기본적인 통신 성능 및 지연 시간 확인.
- **시나리오 B (1v50)**: 단일 컨트롤러가 50개의 비동기 연결을 얼마나 효율적으로 처리하는지 측정.

---

## 3. 성능 측정 스크립트

### 3.1 통합 부하 테스트 실행 (`test_epoll_scale.sh`)
```bash
# 50대의 에이전트를 기동하여 부하 생성
chmod +x test_epoll_scale.sh
./test_epoll_scale.sh 50
```
### 3.2 리소스 모니터링 및 안정화 측정
시스템 기동 직후의 오버헤드를 제외하고 정확한 정상 상태(Steady State)를 측정하기 위해, **최소 30초 이상** 부하를 유지한 뒤 데이터를 수집한다.

```bash
# 30초 대기 후 컨트롤러의 CPU/MEM 점유율 1회 측정
sleep 30
docker stats sv-controller --no-stream --format "CPU={{.CPUPerc}} MEM={{.MemUsage}}"
```

## 6. 단위 테스트 결과 (구현 완료)

### 6.1 Logger (`sv_logger`)
- **싱글톤 검증**: `LoggerFactory`를 통한 유일 객체 보장 확인.
- **로그 레벨**: 설정된 레벨(`WARN` 등)에 따른 출력 필터링 확인.
- **안정성**: `LOG_XXXX` 매크로 연속 호출 시 크래시 없음 확인.

### 6.2 Memory Pool (`sv_core`) - [진행 중]
